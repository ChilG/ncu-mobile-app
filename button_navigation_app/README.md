# Workshop: การสร้างปุ่มและการนำทางใน Flutter (button_navigation_app)

แอปพลิเคชันสำหรับสาธิตการสร้างปุ่มแบบต่าง ๆ ใน Flutter ได้แก่ `TextButton`, `OutlinedButton` และ `ElevatedButton` พร้อมแสดงวิธีการใช้ `Navigator` ในการเปิดหน้าจอใหม่และย้อนกลับมายังหน้าจอหลัก

---

## 1. เปรียบเทียบความแตกต่างและการใช้งานของปุ่มทั้ง 3 ประเภท

ในระบบ Material Design ของ Flutter มีปุ่มหลัก 3 ประเภทที่ถูกออกแบบมาเพื่อให้มีระดับความสำคัญทางสายตา (Visual Emphasis) ที่ต่างกัน:

| ประเภทของปุ่ม      | รูปลักษณ์ (Appearance)                                       | กรณีใช้งานที่เหมาะสม (Appropriate Use Cases)                                                                                         | ระดับความสำคัญ (Emphasis)          |
| :----------------- | :----------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------- | :--------------------------------- |
| **ElevatedButton** | มีพื้นผิว "ยกขึ้น" จากพื้นหลัง มีเงา (Shadow) ให้มิตินูนเด่น | เหมาะสำหรับ Action หลักที่สำคัญที่สุดในหน้าจอ เช่น ปุ่ม "บันทึก", "ยืนยันการสั่งซื้อ", "ส่งข้อมูล"                                   | **ระดับสูง (High Emphasis)**       |
| **OutlinedButton** | มีเส้นขอบรอบตัวปุ่ม ไม่มีพื้นหลังทึบ (พื้นหลังโปร่งใส)       | เหมาะสำหรับ Action รองลงมาที่ต้องการความชัดเจนในการมองเห็นกรอบ เช่น ปุ่ม "ยกเลิก", "กลับไป", "แชร์ข้อมูล"                            | **ระดับปานกลาง (Medium Emphasis)** |
| **TextButton**     | มีเพียงข้อความเท่านั้น ไม่มีกรอบหรือพื้นหลังยกตัวขึ้น        | เหมาะสำหรับ Action เสริม หรือส่วนเสริมที่กลมกลืนกับพื้นหลัง เช่น เมนูบนการ์ด (Card), ปุ่มในกล่องข้อความเตือน (Dialog), หรือปุ่มลิงก์ | **ระดับต่ำ (Low Emphasis)**        |

## 2. การทำงานของ `Navigator.push` ในการเปลี่ยนหน้าจอ

ใน Flutter, `Navigator` ทำหน้าที่จัดการเส้นทางหน้าจอ (Routing) ในลักษณะเป็นโครงสร้างข้อมูลแบบ Stack (First-In, Last-Out) โดยเมื่อต้องการเปลี่ยนหน้าไปยังหน้าใหม่ จะใช้คำสั่ง `Navigator.push`

### ขั้นตอนการทำงานของ `Navigator.push`:

1. **การเรียกใช้คำสั่ง**: เมื่อผู้ใช้ tap ปุ่มที่กำหนด จะมีการเรียกใช้ฟังก์ชัน `Navigator.push(context, route)`
2. **การระบุเส้นทาง (Route)**: ใช้ `MaterialPageRoute` เพื่อเป็นตัวกำหนดเส้นทาง (Route) และสร้าง Widget หน้าจอใหม่ผ่าน `builder` เช่น:
   ```dart
   Navigator.push(
     context,
     MaterialPageRoute(builder: (context) => const TextPage()),
   );
   ```
3. **การ Push เข้าสู่ Stack**: Flutter จะนำเอาหน้าจอใหม่ (เช่น `TextPage`) ไปซ้อนทับไว้ด้านบนสุดของ Stack (มองเห็นเป็นหน้าจอปัจจุบัน) พร้อมกับแสดง Animation การเลื่อนหน้าจอตามแพลตฟอร์ม (iOS เลื่อนจากขวา, Android เลื่อนขึ้น)
4. **การกลับสู่หน้าเดิม**: เมื่อต้องการปิดหน้าจอปัจจุบันและย้อนกลับ จะใช้คำสั่ง `Navigator.pop(context)` ซึ่งจะเป็นการดึง (Pop) หน้าจอด้านบนสุดออกจาก Stack ทำให้หน้าจอด้านล่างถัดไปกลับมาแสดงผลตามเดิม
